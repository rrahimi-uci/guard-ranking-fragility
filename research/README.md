# research/ — reproduction bundle for the paper

Bundle for *"The Benchmark Chooses the Winner: Operating Points, Fine-Tuning, and the Fragility of Safety-Guard Rankings."* The bundle is self-contained in **code**; the evaluation scripts reconstruct their test rows from public Hugging Face datasets (see **Caveats**).

```
research/
  paper/          ACM paper: benchmark_chooses_the_winner.{tex,pdf}, refs.bib, figures/, tables/
    Makefile, README.md, DRAFT.md, metrics_survey.md
  scripts/        the paper's producing scripts (see scripts/README.md)
  notebooks/
    paper_reproduction.ipynb          CANONICAL: one Run-All entrypoint (orchestrates the scripts)
    smollm3_guard_reproduction.ipynb   companion DEMO — NOT the paper's study (see below)
    data/         bundled benchmarks (incl. guard_benchmark_hard.jsonl, mortgage_split.json)
    outputs/      score caches + summary_*.json  (gitignored / derived — regenerated by the scripts)
    README.md, .env
  scripts/        the paper's producing scripts (see scripts/README.md)
  docs/           supporting design/results docs
  requirements.txt  pinned Python deps
  .env.example
```

## Reproduce in one notebook (canonical)
`notebooks/paper_reproduction.ipynb` is the single Run-All entrypoint. It does not reimplement the paper — it **orchestrates the scripts below** (so its logic can't drift) and renders every table/figure from the JSON they emit, with a final validation checklist. Two modes via the `REPRO_MODE` env var:
- `verify_cached` (default) — load existing `outputs/*.json` and render tables/figures fast.
- `recompute` — run each script to (re)produce the JSON, then render. **Heavy**: needs a GPU (local guards), `OPENAI_API_KEY` (GPT), network (open guards), and time.
```bash
cd research
python -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt
cp .env.example .env            # optional: OPENAI_API_KEY (gpt baseline), HF_TOKEN (gated models)
REPRO_MODE=verify_cached jupyter nbconvert --to notebook --execute notebooks/paper_reproduction.ipynb --output run.ipynb
# full recompute (heavy): REPRO_MODE=recompute jupyter nbconvert --execute ...
```

## Or run the producing scripts directly (from `research/`)
```bash
python scripts/eval_corrected.py         # in-house pool: AUPRC / matched-FPR / calibration
python scripts/eval_novel_gaps.py        # novel held-out (OOD)
python scripts/verify_novel.py           # re-ground the novel aggregate
python scripts/score_base_inhouse.py     # base scores on the in-house rows
python scripts/recompute_base_vs_tuned.py# base-vs-tuned decomposition
python scripts/eval_mortgage_hard.py     # mortgage case study (334-row hardened benchmark)
python scripts/reground_gpt_inhouse.py   # in-house GPT baseline, abstain policy
```
See `scripts/README.md` for the full index. Outputs land in `notebooks/outputs/nb-smollm3-guard/` (gitignored).

## Companion demo notebook (not the paper)
`notebooks/smollm3_guard_reproduction.ipynb` is a self-contained end-to-end SmolLM3-3B guard demo (train → calibrate → evaluate against mini-judge baselines). It is a **different experiment** from the paper's open-guard / AUPRC / matched-FPR study, and its GPT baseline uses a demo-grade fail-closed default (the paper's abstain policy lives in the eval scripts). Treat it as an illustrative pipeline, not the paper's reproduction path.

## Build the paper
`paper/` compiles standalone (figures are committed vector PDFs):
```bash
cd research/paper && make        # tectonic -> benchmark_chooses_the_winner.pdf
```
`make figures` re-runs `scripts/make_figures.py`, which draws the figures from **inline, hand-entered** values (it does not read the result JSON) — so keep it in sync with the TeX tables by hand.

## Caveats (reproducibility)
- **Data is reconstructed live.** `eval_corrected.py` and `verify_novel.py` rebuild their rows from Hugging Face datasets **without pinned revisions**, so exact counts (2,018 / 2,020) and row alignment can drift if upstream data changes. Pin dataset revisions or snapshot the rows locally for byte-exact reruns.
- **Outputs are derived.** `notebooks/outputs/` is gitignored; regenerate it by running the scripts above.
- **Figures are hand-grounded.** `make_figures.py` hard-codes the reported numbers inline; it is not provenance-linked to the JSON.
- **Hardened mortgage benchmark.** The version the paper analyzes is `notebooks/data/benchmarks/full/guard_benchmark_hard.jsonl` (334 trap-typed items). The `wf_build_hard_benchmark*.mjs` / `build_hard_jsonl.mjs` scripts are an **earlier, superseded** synthetic minimal-pair builder and do **not** regenerate that file.
